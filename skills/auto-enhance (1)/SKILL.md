# 자동 강화 엔진

세션 활동 데이터를 분석하여 스킬과 훅의 개선점을 찾고, 사용자 승인 후 적용합니다.

## 실행 시점

- 주기적으로 (주 1회 권장)
- 새 스킬을 많이 추가한 후
- "뭔가 자동화할 수 있을 것 같은데" 느낌이 들 때
- 세션 시작 시 패턴 로그에 "자동 강화 권장" 메시지가 표시될 때

## 데이터 소스

| 파일 | 내용 |
|------|------|
| `~/.claude/logs/sessions.jsonl` | 세션별 활동 기록 (생성 파일, 사용 스킬) |
| `~/.claude/logs/patterns.jsonl` | 반복 패턴 감지 결과 |
| `~/.claude/skill-registry.json` | 현재 등록된 스킬 목록 |
| `~/.claude/settings.json` | 현재 훅 설정 |
| `~/.claude/output/` | 생성된 결과물 |

## 워크플로우

### Step 1: 데이터 수집 및 현황 파악

```bash
# 세션 로그 최근 30일
tail -100 ~/.claude/logs/sessions.jsonl 2>/dev/null

# 패턴 로그
cat ~/.claude/logs/patterns.jsonl 2>/dev/null

# 현재 스킬 목록
cat ~/.claude/skill-registry.json 2>/dev/null

# 현재 훅 설정
cat ~/.claude/settings.json | python3 -c "import sys,json; d=json.load(sys.stdin); print(json.dumps(d.get('hooks',{}), indent=2, ensure_ascii=False))"
```

현황을 표시합니다:

```markdown
## 자동 강화 현황 분석

**분석 기간:** 최근 30일
**총 세션 수:** N
**생성된 결과물:** N개

### 스킬 사용 빈도
| 스킬 | 사용 횟수 | 최근 사용 |
|------|----------|----------|
| ... | ... | ... |

### 결과물 유형 분포
| 유형 | 건수 | 비율 |
|------|------|------|
| 상세페이지 | N | % |
| 기획전 | N | % |
| 기타 | N | % |
```

### Step 2: 개선점 분석

5가지 영역에서 개선점을 분석합니다:

#### 2a. 미사용 스킬 감지
- 레지스트리에 등록되어 있지만 30일간 한 번도 사용되지 않은 스킬
- **제안**: 삭제 또는 통합

#### 2b. 반복 작업 패턴
- 세션 로그에서 동일 유형 output이 3회 이상 반복
- 스킬 없이 수동으로 반복되는 작업
- **제안**: 새 스킬 생성 또는 기존 스킬에 템플릿 추가

#### 2c. 훅 효율성
- enhance_prompt.py의 키워드가 최근 프롬프트 패턴을 반영하는지
- 훅 실행 시간이 과도하지 않은지
- **제안**: 키워드 추가/제거, 타임아웃 조정

#### 2d. 스킬 간 중복/갭
- 비슷한 역할의 스킬이 여러 개 있는지
- 커버되지 않는 업무 영역이 있는지
- **제안**: 스킬 통합 또는 새 스킬 생성

#### 2e. 지식 베이스 최신성
- `~/.claude/knowledge/` 파일의 최근 수정일 확인
- 시즌 캘린더가 현재 시즌을 반영하는지
- **제안**: 지식 베이스 업데이트

### Step 3: 제안 보고서

```markdown

| # | 영역 | 제안 | 영향도 |
|---|------|------|--------|
| 1 | ... | ... | 높음/중간/낮음 |

### 사용자 확인 필요
| # | 영역 | 제안 | 사유 |
|---|------|------|------|
| 1 | ... | ... | ... |

### 장기 개선 (백로그)
| # | 영역 | 아이디어 | 예상 효과 |
|---|------|---------|----------|
| 1 | ... | ... | ... |
```

### Step 4: 적용 (`$ARGUMENTS`에 따라)

**"분석만"** (기본): Step 3까지만 실행, 보고서만 출력

**"적용"**: 사용자가 승인한 "즉시 적용" 항목을 실행:

1. **enhance_prompt.py 키워드 갱신**
   - 새 키워드를 `base_keywords` 리스트에 추가
   - 새 컨텍스트 규칙을 조건문에 추가

2. **스킬 레지스트리 재동기화**
   ```bash
   bash ~/.claude/hooks/sync-skill-registry.sh
   ```

3. **불필요 스킬 정리 제안**
   - 삭제 대상 스킬 목록을 AskUserQuestion으로 확인
   - 승인된 것만 삭제

4. **새 스킬 생성**
   - 반복 패턴에서 도출된 새 스킬을 `~/.claude/skills/` 에 생성
   - SKILL.md 표준 형식 준수
   - 생성 후 레지스트리 재동기화

**"전체"**: "적용" + 장기 개선 항목을 태스크로 등록:
   - `~/.claude/tasks/jjang-md-team/` 에 태스크 파일 생성

### Step 5: 변경 로그

적용된 모든 변경을 기록합니다:

```bash
# 변경 로그에 추가
echo "[$(date +%Y-%m-%d)] auto-enhance: <변경 요약>" >> ~/.claude/logs/enhance-history.log
```

```markdown
## 적용 완료

### 변경사항
| # | 유형 | 내용 | 파일 |
|---|------|------|------|
| 1 | 스킬 생성 | ... | ~/.claude/skills/... |
| 2 | 키워드 추가 | ... | enhance_prompt.py |
| 3 | 훅 수정 | ... | settings.json |

### 검증
- 스킬 레지스트리: N개 (이전: M개)
- enhance_prompt.py: 키워드 N개 (이전: M개)
- 훅 수: N개 (변경 없음/+M)
```

## 자동 강화 우선순위 기준

| 우선순위 | 조건 | 예시 |
|---------|------|------|
| 높음 | 주 3회 이상 반복 + 스킬 없음 | 매번 수동으로 가격 조사 |
| 중간 | 기존 스킬이 있지만 비효율 | 스킬 실행 후 항상 수동 보정 |
| 낮음 | 가끔 발생 + 영향 적음 | 월 1회 하는 작업 |

## 예외사항

1. **settings.json 직접 수정은 사용자 확인 필수** - 훅 변경은 시스템 전체에 영향
2. **스킬 삭제 시 반드시 확인** - 다른 사용자/PC에서 사용 중일 수 있음
3. **enhance_prompt.py 수정 후 반드시 문법 검증**: `python3 -c "import py_compile; py_compile.compile('enhance_prompt.py')"`
4. **30일 미만 데이터로는 장기 제안 보류** - 데이터 부족 시 "분석만" 모드로 전환
