# 재고 체크

고도몰 API를 호출하여 지정 상품의 옵션별 재고를 조회하고, 결과를 Gmail로 발송하는 커맨드입니다.

## 입력
- `/재고체크` - 설정 파일의 전체 상품 재고 확인
- `/재고체크 1000000001` - 특정 상품번호만 확인
- `/재고체크 부족만` - 임계값 이하 상품만 표시

인자: $ARGUMENTS

## 실행 절차

### 1단계: 설정 로드
`~/.claude/config/stock-monitor.json` 파일을 읽어 아래 정보를 확인한다:
- `godomall.api_url` — 고도몰 API 기본 URL
- `godomall.partner_key` — 파트너 인증 키
- `monitor_products` — 모니터링 대상 상품 목록 (goodsNo, name, alert_threshold)
- `email` — 메일 발송 설정

**검증**: partner_key가 `여기에_파트너키_입력`이면 사용자에게 키 입력을 요청하고 중단한다.

### 2단계: 고도몰 API 호출 — 상품별 옵션/재고 조회

대상 상품 각각에 대해 **WebFetch** 도구로 고도몰 API를 호출한다.

#### API 엔드포인트
```
GET {api_url}/goods/{goodsNo}
Header: Authorization: Bearer {partner_key}
Header: Content-Type: application/json
```

#### 대안 엔드포인트 (위가 안 될 경우)
고도몰5 버전에 따라 아래 형태일 수 있다:
```
GET {api_url}/goods/goods_view.php?goodsNo={goodsNo}&partnerKey={partner_key}
```
또는
```
POST {api_url}/partner/goods.php
Body: { "partnerKey": "{partner_key}", "mode": "get", "goodsNo": "{goodsNo}" }
```

API 응답 실패 시, 에러 메시지를 기록하고 다음 상품으로 넘어간다.

#### 응답에서 추출할 정보
- 상품명 (goodsNm)
- 옵션 목록 (optionName, optionValue)
- 옵션별 재고 수량 (stockCnt)
- 판매 상태 (goodsStatus / soldOut)

### 3단계: 재고 현황 정리

조회 결과를 아래 형식으로 정리한다:

#### 전체 재고 현황표
| 상품번호 | 상품명 | 옵션 | 재고 | 상태 |
|---------|--------|------|------|------|
| goodsNo | goodsNm (풀네임, 절대 줄여쓰지 않음) | 옵션값 | 수량 | 정상/부족/품절 |

**중요**: 상품명은 반드시 전체 풀네임으로 표시한다. 줄임말이나 약칭 사용 금지.

#### 상태 판정 기준
- **품절**: 재고 0
- **부족**: 재고 > 0 이고 alert_threshold 이하
- **정상**: 그 외

#### 요약
- 전체 모니터링 상품 수
- 정상 / 부족 / 품절 각 수량
- 부족+품절 상품이 있으면 긴급 표시

### 4단계: 메일 발송

Gmail MCP 도구(`mcp__claude_ai_Gmail__gmail_create_draft`)를 사용하여 결과를 메일로 발송한다.

#### 메일 구성
- **제목**: `[짱베이스볼 재고알림] 2026-03-14 재고 현황 (부족 N건 / 품절 N건)`
- **받는 사람**: 설정의 send_to가 "self"이면 내 Gmail 주소로 발송
- **본문**: 아래 HTML 형식

```
재고 모니터링 결과 ({현재 날짜시간})

[긴급 알림 섹션 - 부족/품절이 있을 때만]
- 품절 상품 목록
- 재고 부족 상품 목록

[전체 현황표]
위 3단계에서 정리한 표

[요약]
- 정상: N개 / 부족: N개 / 품절: N개

---
짱베이스볼 재고 자동 모니터링
```

#### 메일 발송 방식
- `send_to`가 "self"이면 → `mcp__claude_ai_Gmail__gmail_create_draft`로 초안 생성 후 사용자에게 "초안이 생성되었습니다. Gmail에서 확인 후 발송하세요" 안내
- 사용자가 "바로 보내줘"라고 하면 발송까지 진행

### 5단계: 결과 출력

화면에도 요약을 출력한다:

```
## 재고 체크 완료

| 구분 | 수량 |
|------|------|
| 모니터링 상품 | N개 |
| 정상 | N개 |
| 재고 부족 | N개 |
| 품절 | N개 |

[부족/품절 상품이 있으면 상세 목록 표시]

Gmail 초안이 생성되었습니다. 확인 후 발송하세요.
```

## 에러 처리
- API 키 미설정 → 키 입력 안내 후 중단
- API 호출 실패 → 해당 상품 건너뛰고 에러 기록, 나머지 계속 진행
- 모니터링 상품 0개 → 설정 파일에 상품 추가 안내
- Gmail MCP 연결 실패 → 화면 출력만 진행, 메일 발송 스킵

## 설정 파일 위치
`~/.claude/config/stock-monitor.json`

### 설정 예시
```json
{
  "godomall": {
    "api_url": "https://jjangbaseball.com/api",
    "partner_key": "실제_파트너키"
  },
  "monitor_products": [
    { "goodsNo": "1000000001", "name": "미즈노 글로브", "alert_threshold": 3 },
    { "goodsNo": "1000000002", "name": "롤링스 배팅장갑", "alert_threshold": 5 }
  ],
  "email": {
    "send_to": "self",
    "subject_prefix": "[짱베이스볼 재고알림]"
  },
  "alert_threshold_default": 5
}
```

## 출력 원칙
- 표 형식으로 한눈에 파악 가능
- 부족/품절은 상단에 긴급 표시
- 실행 가능한 다음 액션 제안 (발주, 품절 처리 등)
