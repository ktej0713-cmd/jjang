# 상세페이지 HTML 생성

짱베이스볼 상품 상세페이지를 **고도몰 + 네이버 스마트스토어 공용** HTML로 생성합니다.

인자: $ARGUMENTS

---

## 인자 파싱

`$ARGUMENTS`에서 아래 형식을 먼저 파싱한다.

```
/상세페이지 [URL 또는 상품명] --제외=섹션1,섹션2
```

**`--제외` 옵션 (선택):**
- `--제외=비교표` → 비교표 섹션 생략
- `--제외=배송` → 배송 안내 생략
- `--제외=포지션` → 포지션별 추천 생략
- `--제외=사이즈` → 사이즈 가이드 생략
- `--제외=비교표,배송` → 복수 지정 가능

`--제외` 옵션이 없으면 모든 섹션을 기본 포함한다.

- 배팅장갑/수비장갑 → 장갑

#### 디자인 규칙 (고도몰 + 스마트스토어 공용 필수)

**플랫폼 호환 원칙**: 스마트스토어 기준을 충족하면 고도몰도 자동 충족된다.

**절대 금지 (스마트스토어 에디터가 제거/무력화)**
- `<style>` 태그 금지 — 모든 스타일은 `style=""` 인라인으로만
- `class=""` 속성 금지 — 인라인 스타일로 대체
- `<details>` / `<summary>` 금지 — 항상 펼침 Q/A `<div>`로 대체
- `::before` / `::after` 의사요소 금지 — `<span>` + HTML 엔티티로 대체
- `display:flex` / `display:grid` 금지 — `<table>` 레이아웃으로 대체
- `<ul>` / `<li>` 금지 — `<div>` + `<span>` 조합으로 대체
- `<script>` 태그 금지 — JS 의존 기능 사용 불가

**허용 및 사용 방법**
- 폰트: `'Noto Sans KR', -apple-system, BlinkMacSystemFont, sans-serif` (`<link>` CDN)
- 컬러: `#1B2A4A` (primary), `#D4A843` (accent) — 인라인 `style=""` 직접 사용
- 섹션 제목: `style="border-left:4px solid #D4A843; padding-left:16px; font-size:24px; font-weight:700; color:#1B2A4A;"`
- 최대 너비: `style="max-width:860px; margin:0 auto;"` — 래퍼 div에 적용
- 모바일 반응형: `<table>`의 `width` 속성으로 대응 (% 단위)
- 이미지: 860×420px 플레이스홀더 `style="width:100%; background:#2C3E6B; color:#D4A843; text-align:center; padding:80px 0; display:block;"`
- 이모지 사용 금지
- 과장 표현 금지 ("최저가", "파격 할인" 등)

#### GEO/AEO 최적화 (필수 포함)

**JSON-LD Product 스키마** (`<script type="application/ld+json">` — 상단 포함):
```
@type: Product, name, alternateName, description(150자), brand, sku, material, color, size
offers: @type Offer, priceCurrency KRW, price(숫자), availability InStock, seller 짱베이스볼
```

**JSON-LD FAQPage 스키마** (별도 스크립트 — FAQ 섹션이 포함된 경우에만):
- 사이즈 선택 / 소재 / 착용법 / 규정 허용 여부 / 관리법 등 실구매자 질문 4개
- JSON-LD(고도몰용) + 마이크로데이터(스마트스토어/쿠팡 호환) — 중복 아닌 상호보완

**FAQ 마이크로데이터 패턴** (FAQ 섹션 HTML 구조에 항상 포함):

두 가지 구조화 데이터를 동시에 적용한다:
- JSON-LD `FAQPage` 스크립트: 파일 상단에 배치 (고도몰 용)
- 마이크로데이터 속성: FAQ HTML 구조에 직접 삽입 (스마트스토어/쿠팡 호환)

FAQ 섹션 HTML 마이크로데이터 구조:
```html
<!-- FAQ 감싸는 최상위 div -->
<div itemscope itemtype="https://schema.org/FAQPage">

  <!-- 각 Q&A 항목 -->
  <div itemscope itemprop="mainEntity" itemtype="https://schema.org/Question" style="margin-bottom:8px; border:1px solid #DDDDDD;">

    <!-- 질문 -->
    <div itemprop="name" style="padding:14px 16px; background:#F9F9F9; font-weight:700; color:#1B2A4A; font-size:14px;">
      <span style="color:#D4A843; font-weight:700; margin-right:4px;">Q.</span>질문 내용
    </div>

    <!-- 답변 래퍼 -->
    <div itemprop="acceptedAnswer" itemscope itemtype="https://schema.org/Answer">
      <!-- 답변 텍스트 -->
      <p itemprop="text" style="padding:14px 16px; font-size:14px; color:#444444; line-height:1.8; background:#ffffff; margin:0;">
        답변 내용
      </p>
    </div>

  </div>

</div>
```

**상품 정의 문장** (히어로 바로 아래):
- `"[상품명]은 [핵심기능]을 갖춘 [카테고리]입니다."` — 두괄식 1~3문장

### 3단계: 파일 저장

`C:\Users\NO\.claude\output\상세페이지\` 폴더에 **1개 파일만** 저장한다.

**파일명**: `상세_{브랜드}_{상품명}_{날짜}_공용.html`
- 날짜 형식: YYYYMMDD (예: 20260312)
- 공백 제거, 특수문자 없이
- `_공용` 접미사 = 고도몰 + 스마트스토어 모두 사용 가능

**파일 구조** (고도몰/스마트스토어 HTML 소스 모드 공용):
```
<link> (Noto Sans KR Google Fonts)
<script type="application/ld+json"> (Product 스키마)
<script type="application/ld+json"> (FAQPage 스키마 — FAQ 섹션 포함 시만)
<!-- <style> 태그 없음 — 모든 스타일은 인라인 -->
<div style="max-width:860px; margin:0 auto; font-family:'Noto Sans KR',-apple-system,sans-serif; color:#222222; line-height:1.7; font-size:17px;">
  <!-- [SEC:히어로] --> ... <!-- [/SEC:히어로] -->
  <!-- [SEC:필요상황] --> ... <!-- [/SEC:필요상황] -->
  ...각 섹션마다 마커 적용...
</div>
<!-- <script> 태그 없음 — JS 없이 동작하는 구조만 사용 -->
```

`<html>/<head>/<body>` 태그 없음. 모든 스타일 인라인 적용으로 플랫폼 충돌 방지.
