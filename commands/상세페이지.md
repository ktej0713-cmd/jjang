# 상세페이지 HTML 생성

짱베이스볼 상품 상세페이지를 **고도몰 + 네이버 스마트스토어 공용** HTML로 생성합니다.

인자: $ARGUMENTS

---

## 인자 파싱

`$ARGUMENTS`에서 아래 형식을 먼저 파싱한다.

```
/상세페이지 [URL 또는 상품명] --제외=섹션1,섹션2
```

**`--제외` 옵션 (선택):**
- `--제외=비교표` → 비교표 섹션 생략
- `--제외=배송` → 배송 안내 생략
- `--제외=포지션` → 포지션별 추천 생략
- `--제외=사이즈` → 사이즈 가이드 생략
- `--제외=비교표,배송` → 복수 지정 가능

`--제외` 옵션이 없으면 모든 섹션을 기본 포함한다.

---

## 실행 절차 (3단계)

### 1단계: 상품 정보 수집

URL이 제공된 경우 WebFetch로 스크래핑한다. 상품명만 제공된 경우 아래 내장 지식을 사용한다.

**카테고리 판별 기준:**
- 글러브/미트 → 글러브
- 배트 → 배트
- 헬멧/암가드/풋가드/마스크/프로텍터/렉가드 → 보호장비
- 스파이크/인조잔디화/포인트화 → 야구화
- 바지/상의/점퍼/모자/양말 → 의류
- 배팅장갑/수비장갑 → 장갑
- 나머지 → 용품(범용)

---

### 2단계: 상품 데이터 JSON 작성

**중요**: HTML을 직접 Write하지 않는다. 상품 데이터만 JSON으로 작성하면 템플릿 엔진이 HTML을 생성한다.
이렇게 하면 출력 토큰이 ~15KB → ~3KB로 80% 절감된다.

#### 제외 옵션 처리

`--제외`에 지정된 섹션명을 `"exclude"` 배열에 넣는다:
- `--제외=비교표,사이즈` → `"exclude": ["비교표", "사이즈"]`
- 히어로, 스펙은 제외 불가 (항상 생성)

#### 콘텐츠 작성 규칙

- 이모지 사용 금지
- 과장 표현 금지 ("최저가", "파격 할인" 등)
- 킵 레더/세토스티어 등 가죽 소재: "늘어난다" 표현 금지 → "포켓이 형성된다", "손에 적응한다"
- 서머리에 가격 하드코딩 금지 (product.price는 JSON-LD 스키마에만 사용)
- 스펙 수치 근거 없으면 "약" 표기
- FAQ 답변은 실구매자 관점

#### 내장 브랜드/가격 지식 (파일 읽기 불필요)

| 카테고리 | 브랜드 | 가격대 | 타겟 |

### 3단계: HTML 생성 실행

JSON 파일 저장 후, 템플릿 엔진을 실행하여 HTML을 생성한다.

**실행 방법** (둘 중 하나 선택):

```bash
# Bash로 실행
node ~/.claude/scripts/generate-detail-page.js ~/.claude/output/product-data.json
```

또는 `ctx_batch_execute`로 실행 (출력이 컨텍스트에 진입하지 않음):

```
ctx_batch_execute(
  commands: [{"label": "생성", "command": "node ~/.claude/scripts/generate-detail-page.js ~/.claude/output/product-data.json"}],
  queries: ["생성 완료 파일 섹션"]
)
```

**생성 결과**: 스크립트가 `~/.claude/output/{fileName}`에 HTML 파일을 저장하고, 요약만 콘솔에 출력한다.

**템플릿 엔진이 자동 처리하는 항목** (JSON에 넣을 필요 없음):
- 인라인 스타일 (모든 CSS)
- `<table>` 레이아웃 구조
- SEC 마커 (`<!-- [SEC:섹션명] -->`)
- 섹션 배경색 교차 (흰색 ↔ 회색, 전문가만 다크)
- JSON-LD Product 스키마
- JSON-LD FAQPage 스키마 + HTML 마이크로데이터
- Noto Sans KR 폰트 링크
- 채널톡 CTA 버튼 (서머리 하단)
- 고도몰 + 스마트스토어 공용 호환 (flex/grid/class/style태그 없음)

---

## 수정 요청 처리 방법

생성된 파일 수정 요청이 오면:
1. `<!-- [SEC:섹션명] -->` 마커로 해당 섹션 범위를 즉시 특정
2. 해당 범위만 Edit (파일 전체 읽기 불필요)
3. 섹션 삭제 시 `<!-- [SEC:섹션명] -->` ~ `<!-- [/SEC:섹션명] -->` 전체 삭제

## 품질 기준 (생성 중 자체 검증)

### 콘텐츠
- [ ] 야구 커뮤니티에서 신뢰를 잃을 표현 없음
- [ ] 스펙 수치 근거 있음 (추측 시 "약" 표기)
- [ ] 타겟(사회인/유소년/엘리트) 명확히 지정
- [ ] GEO 필수: Product스키마 + 정의문장
- [ ] FAQ 섹션 포함 시: JSON-LD FAQPage 스크립트 + HTML 마이크로데이터 속성 양쪽 모두 포함
- [ ] SEC 마커 모든 섹션에 적용됨
- [ ] 이모지 없음
- [ ] 과장 표현 없음

### 플랫폼 호환 (고도몰 + 스마트스토어 공용)
- [ ] `<style>` 태그 없음 — 모든 스타일 인라인
- [ ] `class=""` 속성 없음
- [ ] `<details>` / `<summary>` 없음 — Q/A 펼침 div 사용
- [ ] `::before` / `::after` 없음 — span + 엔티티 사용
- [ ] `display:flex` / `display:grid` 없음 — table 레이아웃
- [ ] `<ul>` / `<li>` 없음 — div + span 조합
- [ ] `<script>` 태그 없음
- [ ] 파일명 `_공용.html` 접미사
