# 상세페이지 HTML 생성

짱베이스볼 상품 상세페이지를 고도몰 붙여넣기용 HTML로 생성합니다.

인자: $ARGUMENTS

---

## 인자 파싱

`$ARGUMENTS`에서 아래 형식을 먼저 파싱한다.

```
/상세페이지 [URL 또는 상품명] --제외=섹션1,섹션2
```

**`--제외` 옵션 (선택):**
- `--제외=비교표` → 비교표 섹션 생략
- `--제외=배송` → 배송 안내 생략
- `--제외=포지션` → 포지션별 추천 생략
- `--제외=사이즈` → 사이즈 가이드 생략
- `--제외=비교표,배송` → 복수 지정 가능

`--제외` 옵션이 없으면 모든 섹션을 기본 포함한다.

---

## 실행 절차 (3단계)

### 1단계: 상품 정보 수집

URL이 제공된 경우 WebFetch로 스크래핑한다. 상품명만 제공된 경우 아래 내장 지식을 사용한다.

**카테고리 판별 기준:**
- 글러브/미트 → 글러브
- 배트 → 배트
- 헬멧/암가드/풋가드/마스크/프로텍터/렉가드 → 보호장비
- 스파이크/인조잔디화/포인트화 → 야구화
- 바지/상의/점퍼/모자/양말 → 의류
- 배팅장갑/수비장갑 → 장갑
- 나머지 → 용품(범용)

---

### 2단계: HTML 생성

아래 구조로 즉시 HTML을 생성한다. 추가 파일 읽기 없이 이 커맨드의 내용만으로 완성한다.

#### 섹션 구조 (8개)

각 섹션은 반드시 아래 마커로 감싼다. 마커가 있어야 나중에 특정 섹션만 빠르게 수정 가능하다.

```
<!-- [SEC:섹션명] -->
...콘텐츠...
<!-- [/SEC:섹션명] -->
```

| # | 섹션명 | 내용 | --제외 키워드 |
|---|--------|------|-------------|
| 1 | `히어로` | 이미지 플레이스홀더 + 핵심 카피 1줄 + 상품 정의 문장 | 제외 불가 |
| 2 | `필요상황` | "이런 분께 맞습니다" 공감 유도 3~4가지 | `필요상황` |
| 3 | `스펙` | 주요 사양 표 (소재/무게/사이즈/색상/규격) | 제외 불가 |
| 4 | `체감설명` | 실사용자 관점 설명 (스펙 나열 금지) | `체감설명` |
| 5 | `포지션` | 포지션/타겟별 적합도 카드 | `포지션` |
| 6 | `사이즈` | 사이즈 선택 기준, 사이즈표 | `사이즈` |
| 7 | `FAQ` | 자주 묻는 질문 4개, 아코디언 UI | `FAQ` |
| 8 | `추천포인트` | "이런 분께 추천합니다" 명확한 대상 | `추천포인트` |

**`--제외`에 포함된 섹션명은 HTML에서 완전히 생략한다.**

#### 디자인 규칙

- 폰트: `'Noto Sans KR', -apple-system, BlinkMacSystemFont, sans-serif` (Google Fonts CDN 포함)
- 컬러: `--primary: #1B2A4A`, `--accent: #D4A843`
- 섹션 제목: `style="border-left:4px solid #D4A843; padding-left:16px; font-size:28px; font-weight:700; color:#1B2A4A;"`
- 히어로 타이틀: `font-size:36px; font-weight:800;`
- 본문 기본 크기: `font-size:15px`
- 카드/항목 제목: `font-size:16px; font-weight:700;`
- 서브텍스트: `font-size:13px;`
- 이모지 사용 금지
- 최대 너비 860px (고도몰/네이버 스마트스토어 호환)
- 모바일 반응형 (768px 기준)
- 이미지: `860×420px` 회색 플레이스홀더 (`background:#2C3E6B; color:#D4A843`)
- 과장 표현 금지 ("최저가", "파격 할인" 등)

#### GEO/AEO 최적화 (필수 포함)

**JSON-LD Product 스키마** (`<script type="application/ld+json">` — 상단 포함):
```
@type: Product, name, alternateName, description(150자), brand, sku, material, color, size
offers: @type Offer, priceCurrency KRW, price(숫자), availability InStock, seller 짱베이스볼
```

**JSON-LD FAQPage 스키마** (별도 스크립트 — FAQ 섹션이 포함된 경우에만):
- 사이즈 선택 / 소재 / 착용법 / 규정 허용 여부 / 관리법 등 실구매자 질문 4개

**상품 정의 문장** (히어로 바로 아래):
- `"[상품명]은 [핵심기능]을 갖춘 [카테고리]입니다."` — 두괄식 1~3문장

### 3단계: 파일 저장

`C:\Users\NO\.claude\output\상세페이지\` 폴더에 **1개 파일만** 저장한다.

**파일명**: `상세_{브랜드}_{상품명}_{날짜}_고도몰.html`
- 날짜 형식: YYYYMMDD (예: 20260312)
- 공백 제거, 특수문자 없이

**파일 구조** (고도몰 HTML 소스 모드에 직접 붙여넣기용):
```
<link> (Noto Sans KR Google Fonts)
<script type="application/ld+json"> (Product 스키마)
<script type="application/ld+json"> (FAQPage 스키마 — FAQ 섹션 포함 시만)
<style> (CSS — #jjang-detail 스코핑)
<div id="jjang-detail">
  <!-- [SEC:히어로] --> ... <!-- [/SEC:히어로] -->
  <!-- [SEC:필요상황] --> ... <!-- [/SEC:필요상황] -->
  ...각 섹션마다 마커 적용...
</div>
<script> (아코디언 JS — jjang 접두사)
```

`<html>/<head>/<body>` 태그 없음. CSS와 JS 함수는 모두 `#jjang-detail` 또는 `jjang` 접두사로 스코핑해 고도몰 기본 스타일 충돌 방지.

## 수정 요청 처리 방법

생성된 파일 수정 요청이 오면:
1. `<!-- [SEC:섹션명] -->` 마커로 해당 섹션 범위를 즉시 특정
2. 해당 범위만 Edit (파일 전체 읽기 불필요)
3. 섹션 삭제 시 `<!-- [SEC:섹션명] -->` ~ `<!-- [/SEC:섹션명] -->` 전체 삭제

## 품질 기준 (생성 중 자체 검증)

- [ ] 야구 커뮤니티에서 신뢰를 잃을 표현 없음
- [ ] 스펙 수치 근거 있음 (추측 시 "약" 표기)
- [ ] 타겟(사회인/유소년/엘리트) 명확히 지정
- [ ] GEO 필수: Product스키마 + 정의문장
- [ ] SEC 마커 모든 섹션에 적용됨
- [ ] 이모지 없음
- [ ] 과장 표현 없음
