# 고도몰5 실무 운영 가이드

> 짱베이스볼 일상 운영에서 자주 쓰는 관리자 기능 가이드
> 최종 업데이트: 2026-03-12

---

## 1. 주문 처리 흐름

### 표준 주문 상태 흐름

```
입금대기 → 결제완료 → 상품준비중 → 배송중 → 배송완료 → 구매확정
```

### 주문 상태 코드 전체 목록

| 코드 | 상태 | 설명 |
|------|------|------|
| `o1` | 입금대기 | 가상계좌/무통장 입금 대기 |
| `o2` | 결제완료 | 카드/실시간계좌이체/입금확인 |
| `o3` | 상품준비중 | 출고 준비 중 |
| `o4` | 배송보류 | 배송 보류 상태 |
| `d1` | 배송중 | 운송장 등록 후 배송 시작 |
| `d2` | 배송완료 | 택배사 배송 완료 처리 |
| `p1` | 구매확정 | 구매확정 (적립금 지급) |
| `b1` | 반품접수 | |
| `b2` | 반송중 | 반품 물건 회수 중 |
| `b3` | 반품보류 | |
| `b4` | 반품회수완료 | |
| `c1` | 자동취소 | |
| `c2` | 품절취소 | |
| `c3` | 관리자취소 | |
| `c4` | 고객요청취소 | |
| `e1` | 교환접수 | |
| `e2` | 반송중(교환) | |
| `e3` | 재배송중 | |
| `e4` | 교환보류 | |
| `e5` | 교환완료 | |
| `f1` | 결제시도 | |
| `f2` | 고객결제중단 | |
| `f3` | 결제실패 | |

### 운영 포인트
- **재고 차감 시점**: 기본값 = 배송처리 시 (결제완료 시로 변경 가능)
- **적립금 지급 시점**: 기본값 = 구매확정 시
- 야구용품 특성상 오더 상품(배송 4~6개월)은 `o4 배송보류`로 관리 권장

## 2. 상품 등록 체크리스트

### 최소 등록 요건 (판매 가능 상태)

```
[ ] 상품명 (필수)
[ ] 카테고리 배정
[ ] 판매가 입력
[ ] 재고 설정 (0이면 자동 품절)
[ ] 대표 이미지 등록
[ ] 노출 상태: ON
[ ] 판매 상태: 판매중
```

### 상품 등록 전체 항목 (권장)

| 항목 | 내용 | 비고 |
|------|------|------|
| 상품명 | 검색 키워드 포함 자연스럽게 | 예: "미즈노 프로 하가재팬 1루수 미트 7N형 13인치" |
| 카테고리 | 복수 카테고리 배정 가능 | 주 카테고리 + 시즌 기획전 카테고리 |
| 판매가 | 기본 판매 금액 | |
| 정가(소비자가) | 할인율 표시용 | 입력 시 할인% 자동 계산 |
| 공급가 | 원가 관리용 (비공개) | |
| 재고 | 옵션별 개별 관리 가능 | 0 → 자동 품절 처리 |
| 구매 제한 | 최소/최대 구매수량 | 오더 상품은 1개 제한 권장 |
| 옵션 | 색상/사이즈 등 선택형 | 옵션별 가격 차등 가능 |
| 이미지 | 원본 1000px 이상 권장 | 리스트용/상세용/썸네일 자동 리사이징 |
| 간략 설명 | 상품명 아래 짧은 설명 | 네이버쇼핑 상품명 보조 역할 |
| 상세 설명 | 상세페이지 HTML | PC/모바일 별도 등록 가능 |
| 배송비 | 무료/유료/조건부 | 야구용품 기본: 무료(일정금액 이상) |
| 브랜드 | 브랜드 관리에서 선등록 필요 | |
| 원산지 | 일본산/한국산 등 | 글러브/배트 등 표기 중요 |
| 네이버쇼핑 EP | 네이버 카테고리 ID (8자리) | 반드시 입력 |

## 4. 네이버쇼핑 EP 연동

### EP(상품 DB URL)란?
상품 정보를 담은 피드 URL. 네이버가 하루 **4회** 수집 (01시/10시/14시/18시).

### 설정 방법

**1단계**: 고도몰 관리자에서 상품별 네이버 카테고리 ID(8자리) 입력
```
관리자 > 상품 > 상품 수정 > 하단 > "네이버쇼핑 EP 3.0 정보 등록"
```

**2단계**: 네이버 파트너센터에서 EP URL 등록
```
쇼핑파트너센터 > 상품관리 > 상품정보 수신 현황 > [수정요청] > EP URL 입력
```

**3단계**: 카테고리 ID 확인
```
쇼핑파트너센터 > 상품관리 > "네이버쇼핑 카테고리 ID" 버튼 > 엑셀 다운로드
```

### 짱베이스볼 주요 카테고리 → 네이버 카테고리 매핑

| 짱베이스볼 카테고리 | 네이버 카테고리 |
|------------------|--------------|
| 글러브 (전체) | 스포츠/레저 > 야구/소프트볼 > 글러브/미트 |
| 배트 | 스포츠/레저 > 야구/소프트볼 > 배트 |
| 야구화 | 스포츠/레저 > 야구/소프트볼 > 야구화 |
| 보호장비 | 스포츠/레저 > 야구/소프트볼 > 보호대/헬멧 |
| 유니폼/의류 | 스포츠/레저 > 야구/소프트볼 > 유니폼 |
| 야구공 | 스포츠/레저 > 야구/소프트볼 > 야구공 |

> 정확한 8자리 ID는 파트너센터 카테고리 엑셀에서 확인 필요

## 6. 개발 작업 경로 참조

### module/ 폴더 구조 (커스터마이징 가능 범위)

```
module/
├── Controller/
│   ├── Front/        ← PC 사용자 컨트롤러
│   ├── Mobile/       ← 모바일 사용자 컨트롤러
│   ├── Admin/        ← 관리자 컨트롤러
│   └── Api/          ← API 컨트롤러
├── Component/        ← 비즈니스 로직
│   └── Database/
│       └── DBTableField.php  ← DB 테이블/컬럼 확장
└── Widget/
    └── Front/        ← 위젯
```

### skin 파일 위치

```
data/skin/front/[스킨명]/
```

### 절대 수정 금지 영역

```
Bundle/   ← 고도몰 코어 (자동패치 대상)
core/
vendor/
framework/
```

## 8. 자주 발생하는 운영 이슈

| 이슈 | 원인 | 해결 방법 |
|------|------|---------|
| 네이버쇼핑 미노출 | 카테고리 ID 미입력 | EP 3.0에서 8자리 카테고리 ID 입력 |
| 이미지 느린 로딩 | 원본 이미지 용량 과다 | 1MB 이하로 압축 후 재등록 |
| 재고 불일치 | 재고 차감 시점 설정 | 주문관리 > 재고 차감 시점 확인 |
| 스킨 수정 후 미반영 | 캐시 문제 | 관리자에서 캐시 초기화 |
| 모바일 레이아웃 깨짐 | viewport 미설정 | 스킨 헤더에 viewport meta 태그 확인 |
| 배송비 이중 부과 | 배송비 정책 충돌 | 기본설정 > 배송 정책 재확인 |

## 9. DB 테이블 확장 패턴 (커스텀 컬럼 추가)

```php
// module/Component/Database/DBTableField.php

namespace Component\Database;

class DBTableField extends \Bundle\Component\Database\DBTableField
{
    // 기존 es_goods 테이블에 커스텀 컬럼 추가
    public static function tableGoods($conf = null)
    {
        $arrField = parent::tableGoods($conf);
        // 짱베이스볼 전용 필드 추가
        $arrField[] = ['val' => 'jjangCustomField', 'typ' => 's', 'def' => ''];
        return $arrField;
    }
}
```

**규칙**:
- Engine: InnoDB 필수
- Character Set: utf8mb4
- Prefix: es_/zz_ 제외한 자체 접두사 사용
- 기존 테이블 컬럼 수정/삭제 절대 금지

## 10. 스킨 패치 적용 절차 (수동)

1. NHN커머스 사이트 로그인 → 내 쇼핑몰 관리
2. 패치&업그레이드 메뉴 선택
3. 적용할 패치 게시글 선택
4. 패치 파일 다운로드
5. 압축 파일 검토 (수정 파일 목록 확인)
6. 로컬에서 내 스킨에 맞게 패치 적용
7. '패치확인하기' 클릭
8. 패치 완료 기록

> 프로그램 패치는 자동 적용, **스킨 패치만 수동** 적용 필요
