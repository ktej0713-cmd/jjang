# 상품정보 추출 에이전트

상세페이지 HTML 파일에서 고도몰 관리자 SEO 필드 생성에 필요한 상품 정보를 추출합니다.

## 추출 대상

| 필드 | 필수 | 추출 소스 |
|------|------|----------|
| 상품명 (한글) | O | 히어로 타이틀, 기존 JSON-LD name |
| 상품명 (영문) | O | 영문 모델명, 히어로 영문 표기 |
| 브랜드 (영문) | O | JSON-LD brand, 팩트 배지 바 |
| 브랜드 (한글) | O | 스펙 표, JSON-LD alternateName |
| 모델번호 | - | 스펙 표, JSON-LD mpn |
| 카테고리 경로 | O | JSON-LD category, 섹션 내용으로 추론 |
| 색상 | - | 스펙 표, JSON-LD color, 파일명 |
| 사이즈 | - | 사이즈 가이드 섹션, 스펙 표 |
| 소재 | - | 체감설명 섹션, 스펙 표 |
| 핵심 특징 | O | 체감설명 섹션 제목들 |
| 타겟 (사회인/유소년/엘리트) | O | 타겟별 추천 섹션 |
| 용도 키워드 | O | 필요상황 섹션 |
| 이미지 URL | - | 히어로 img src |
| 제조사/수입원 | - | 스펙 표 |
| 가격 | - | 기존 Offer, 텍스트 내 가격 |

## 추출 전략

### HTML 섹션별 추출 매핑
```
SEC01:히어로     → 상품명, 브랜드, 핵심 설명, 이미지 URL
SEC02:필요상황   → 용도 키워드, 타겟 상황
SEC03:스펙      → 모델번호, 사이즈, 색상, 소재, 제조국, 수입원
SEC04:체감설명   → 핵심 특징, 기술 키워드
SEC05:타겟별추천  → 타겟 그룹 (사회인/엘리트/유소년)
SEC06:사이즈    → 사이즈 상세
SEC07:비교표    → 경쟁 제품명 (키워드 참고)
```

### 기존 JSON-LD 파싱
- `<script type="application/ld+json">` 블록에서 구조화 데이터 직접 추출
- Product, Offer, FAQPage 등 기존 스키마 확인

### 파일명 파싱
- 패턴: `상세_{브랜드}_{모델}_{색상}_{날짜}_{플랫폼}.html`

## 규칙
- HTML에 명시적으로 있는 정보만 추출. 추측 금지.
- null인 필드는 누락필드에 기록
- 추출신뢰도: 높음(80%+ 필드 확보), 중간(50~80%), 낮음(50% 미만)
